# 网络通信模块

## 概述

网络通信模块提供客户端和商家端之间的 TCP 网络通信功能，实现订单数据、结账请求和菜品更新的实时同步。

## 文件列表

- `networkclient.h` / `networkclient.cpp`：网络客户端
- `networkserver.h` / `networkserver.cpp`：网络服务器

## 功能说明

### 主要功能

1. **TCP 服务器**：
   - 监听客户端连接（默认端口 8888）
   - 接收订单数据
   - 发送确认消息
   - 广播菜品更新通知

2. **TCP 客户端**：
   - 连接到服务器
   - 发送订单数据
   - 发送结账请求
   - 接收服务器响应

3. **消息协议**：
   - 使用 JSON 格式传输数据
   - 支持多种消息类型
   - 错误处理和重连机制

## 消息协议

### 消息格式

所有消息使用 JSON 格式，通过 TCP 传输。

### 客户端发送的消息

#### 1. 订单消息 (order)

```json
{
  "type": "order",
  "order": {
    "orderNo": "0210001",
    "username": "顾客",
    "items": [
      {
        "dish": {
          "id": 1,
          "name": "宫保鸡丁",
          "category": "热菜",
          "price": 28.0
        },
        "quantity": 2
      }
    ],
    "status": "Pending",
    "createTime": "2026-03-04T09:28:19"
  }
}
```

#### 2. 结账请求 (checkout)

```json
{
  "type": "checkout",
  "order": {
    "orderNo": "0210001",
    "totalAmount": 56.0
  }
}
```

#### 3. 服务请求 (service)

```json
{
  "type": "service",
  "request": "refresh_dishes"
}
```

### 服务器发送的消息

#### 1. 订单确认 (order_confirm)

```json
{
  "type": "order_confirm"
}
```

#### 2. 结账成功 (checkout_success)

```json
{
  "type": "checkout_success",
  "totalAmount": 56.0
}
```

#### 3. 菜品更新通知 (dish_update)

```json
{
  "type": "dish_update"
}
```

#### 4. 订单完成通知 (order_completed)

```json
{
  "type": "order_completed",
  "orderNo": "0210001",
  "message": "您的订单 0210001 已完成！"
}
```

## 使用流程

### 服务器启动流程

1. 创建 `NetworkServer` 对象
2. 调用 `startServer(8888)` 启动服务器
3. 监听 `QHostAddress::Any`（所有网络接口）
4. 等待客户端连接

### 客户端连接流程

1. 创建 `NetworkClient` 对象
2. 调用 `connectToServer("localhost", 8888)` 连接服务器
3. 等待连接成功信号
4. 连接成功后可以发送消息

### 订单传输流程

1. **客户端发送订单**：
   - 调用 `sendOrder(orderJson)`
   - 将订单数据封装为 JSON
   - 通过 TCP 发送

2. **服务器接收订单**：
   - `onClientReadyRead()` 接收数据
   - 解析 JSON 数据
   - 验证消息类型和格式
   - 发出 `newOrderReceived` 信号

3. **服务器发送确认**：
   - 创建确认消息 JSON
   - 发送给客户端

4. **客户端接收确认**：
   - `onReadyRead()` 接收数据
   - 解析 JSON 数据
   - 发出 `orderConfirmed` 信号

### 菜品更新广播流程

1. 商家端修改菜品数据
2. 调用 `broadcastDishUpdate()`
3. 服务器创建更新消息
4. 遍历所有连接的客户端
5. 向每个客户端发送更新消息
6. 客户端接收并处理更新

### 订单完成通知流程

1. **商家端确认完成订单**：
   - 商家在订单列表中点击"确认完成"按钮
   - 订单状态更新为"已完成"
   - 调用 `sendOrderCompletedNotification(orderNo)`

2. **服务器广播通知**：
   - 创建订单完成消息 JSON（包含 `type: "order_completed"`、订单号和提示消息）
   - 调用 `broadcastMessage()` 广播给所有客户端

3. **客户端接收通知**：
   - `NetworkClient::onReadyRead()` 接收数据
   - 解析 JSON，识别 `order_completed` 类型
   - 发出 `messageReceived` 信号

4. **客户端显示提示**：
   - `ClientMainWindow::onMessageReceived()` 处理消息
   - 显示消息框提示："您的订单 XXX 已完成！"
   - 在状态栏显示提示信息

## 错误处理

### 连接错误

- **连接被拒绝**：服务器未启动或端口被占用
- **远程主机关闭**：服务器主动关闭连接
- **找不到主机**：主机地址错误或网络不通
- **其他错误**：显示通用错误信息

### 数据错误

- **JSON 解析失败**：数据格式错误，忽略该消息
- **缺少必需字段**：数据不完整，忽略该消息
- **消息类型未知**：不支持的消息类型，忽略该消息

### 重连机制

- 客户端连接失败时不影响本地功能
- 订单数据会保存到本地文件
- 可以手动重连或自动重连（可扩展）

## 网络配置

### 默认配置

- **服务器端口**：8888
- **客户端地址**：localhost
- **协议**：TCP
- **数据格式**：JSON

### 修改配置

#### 修改服务器端口

在 `networkserver.cpp` 中：
```cpp
m_networkServer->startServer(9999); // 改为 9999
```

在 `networkclient.cpp` 中：
```cpp
m_networkClient->connectToServer("localhost", 9999);
```

#### 修改客户端地址

在 `networkclient.cpp` 中：
```cpp
m_networkClient->connectToServer("192.168.1.100", 8888);
```

## 性能优化

### 连接管理

- 服务器支持多客户端同时连接
- 使用 `QVector<QTcpSocket*>` 管理客户端列表
- 客户端断开时自动清理

### 数据发送

- 使用 `QJsonDocument::toJson()` 序列化
- 直接通过 `QTcpSocket::write()` 发送
- 不等待确认（异步发送）

### 数据接收

- 使用 `readAll()` 读取所有可用数据
- 假设每次接收完整 JSON 消息
- 如果消息较大，可能需要分片接收（当前未实现）

## 安全考虑

### 当前实现

- 无身份验证
- 无数据加密
- 无访问控制

### 建议改进

1. **身份验证**：
   - 添加登录验证
   - 使用 Token 机制

2. **数据加密**：
   - 使用 TLS/SSL 加密传输
   - 或使用应用层加密

3. **访问控制**：
   - IP 白名单
   - 连接数限制
   - 速率限制

## 注意事项

1. **端口占用**：确保端口 8888 未被其他程序占用
2. **防火墙**：确保防火墙允许端口 8888 的通信
3. **数据完整性**：当前实现假设每次接收完整消息，大消息可能需要分片处理
4. **错误恢复**：网络错误时不影响本地功能，数据会保存到文件
5. **并发处理**：服务器在主线程中处理，大量连接可能影响 UI 响应

## 扩展建议

1. 添加自动重连机制
2. 添加消息队列（离线消息）
3. 添加心跳机制（保持连接）
4. 支持消息分片和重组
5. 添加连接状态监控
6. 实现 TLS/SSL 加密
7. 添加身份验证机制
