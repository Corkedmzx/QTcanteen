# 餐厅点餐系统 (QTcanteen)

一个基于 Qt 框架开发的餐厅点餐管理系统，支持客户端点餐和商家端管理功能。

## 项目简介

QTcanteen 是一个基于 Qt 框架开发的餐厅点餐管理系统，提供客户端点餐和商家端管理功能。

### 当前已实现功能

- **客户端点餐**：顾客可以浏览菜品、下单、查看订单详情
- **商家端管理**：商家可以管理菜品、处理订单、查看订单状态
- **人脸识别登录**：支持 OpenCV 和 SeetaFace 两种人脸识别方案
- **网络通信**：客户端和商家端通过 TCP 网络实时同步订单数据
- **数据持久化**：使用 JSON 文件存储用户、菜品和订单数据

### 功能状态

- ✅ **已完成**：核心点餐功能、订单管理、人脸识别、网络通信
- ⚠️ **待完善**：多平台适配、登录方式扩展、支付模块（需要自行实现）

## 技术栈

- **框架**：Qt 6.x / Qt 5.x
- **语言**：C++17
- **构建系统**：CMake
- **人脸识别**：OpenCV / SeetaFace6Open
- **网络通信**：Qt Network (TCP)
- **数据存储**：JSON 文件

## 快速开始

### 1. 克隆项目

```bash
git clone <repository-url>
cd QTcanteen
```

### 2. 配置 OpenCV（可选）

如果使用 OpenCV 进行人脸识别，需要：

1. 下载并解压 OpenCV 到指定目录（默认：`E:/share/opencv452`）
2. 在 `CMakeLists.txt` 中修改 `OPENCV_ROOT` 路径
3. 编译时 CMake 会自动查找 OpenCV

### 3. 编译项目

```bash
mkdir build
cd build
cmake ..
cmake --build .
```

### 4. 添加菜品图片（可选）

项目中的 `images/` 目录为空，您需要自行添加菜品图片：

1. 将菜品图片文件（支持 .jpg, .jpeg, .png 等格式）放入 `images/` 目录
2. 在商家端添加菜品时，图片路径格式为：`images/图片文件名.jpg`
3. 示例：`images/kungpao.jpg`

**注意**：`images/` 目录中的图片文件不会被提交到 Git 仓库，每个开发者需要自行添加。

### 5. 运行程序

```bash
./QTcanteen  # Linux/Mac
QTcanteen.exe  # Windows
```

## 主要功能

### 客户端功能
- 浏览菜品（按分类显示）
- 添加菜品到购物车
- 提交订单
- 查看订单详情
- 结账功能
- 自动连接商家端服务器

### 商家端功能
- 管理菜品（添加、编辑、删除）
- 查看所有订单
- 更新订单状态（待处理、处理中、已完成、已取消）
- 人脸录入功能
- 启动网络服务器（端口 8888）
- 自动刷新订单列表

### 人脸识别功能
- 支持两种识别方案：
  - **OpenCV**：使用 LBPH 算法或 DNN 深度特征
  - **SeetaFace**：使用 SeetaFace6Open SDK
- 实时人脸检测和跟踪
- 人脸录入和训练
- 人脸识别登录

### 网络通信
- TCP 服务器/客户端架构
- 实时订单同步
- 订单确认机制
- 菜品更新广播

## 数据存储

所有数据存储在 JSON 文件中：
- `user.json`：用户账号和密码
- `dishes.json`：菜品信息
- `orders.json`：订单记录
- `face_model/`：人脸识别模型数据

## 配置说明

### 网络端口
- 默认服务器端口：**8888**
- 可在代码中修改（`networkserver.cpp` 和 `networkclient.cpp`）

### OpenCV 路径
- 默认路径：`E:/share/opencv452`
- 修改 `CMakeLists.txt` 中的 `OPENCV_ROOT` 变量

### SeetaFace 路径
- 默认路径：`E:/share/SeetaFace`
- 可通过环境变量 `SEETAFACE_ROOT` 设置

## 使用说明

### 首次使用

1. 启动程序，进入登录窗口
2. 选择"点餐"可直接进入客户端（无需登录）
3. 选择"商家登录"进入商家端
4. 默认商家账号：`admin`（密码在首次运行时创建）

**重要提示**：
- 当前版本仅支持**账号密码登录**和**人脸识别登录**
- 如需扫码登录、微信登录等现代登录方式，请参考"开发说明"部分自行实现
- 结账功能**不包含实际支付**，仅记录订单状态。如需支付功能，请自行集成支付 SDK

### 商家端操作

1. **录入人脸**：
   - 点击"录入人脸"按钮
   - 选择用户账号
   - 使用摄像头拍照并保存

2. **管理菜品**：
   - 在菜品管理界面添加、编辑或删除菜品
   - 支持设置菜品名称、分类、价格、描述和图片

3. **处理订单**：
   - 在订单列表中查看所有订单
   - 双击订单查看详情
   - 右键菜单更新订单状态

### 客户端操作

1. **浏览菜品**：
   - 按分类查看菜品
   - 点击菜品图片查看详情

2. **下单**：
   - 选择菜品并设置数量
   - 点击"提交订单"
   - 订单会自动发送到商家端

3. **结账**：
   - 在订单详情对话框中点击"结账"
   - 系统会计算总金额并确认
   - **注意**：当前版本不包含实际支付功能，仅记录订单状态。如需支付功能，请参考"开发说明"部分自行实现

## 常见问题

### Q: 编译时找不到 OpenCV？
A: 检查 `CMakeLists.txt` 中的 `OPENCV_ROOT` 路径是否正确，确保 OpenCV 已正确安装。

### Q: 人脸识别功能无法使用？
A: 确保已安装 OpenCV 或 SeetaFace，并在 CMake 配置时正确设置了相关路径。

### Q: 网络连接失败？
A: 确保商家端已启动服务器（端口 8888），客户端会自动尝试连接。

### Q: 数据文件丢失？
A: 程序会在首次运行时创建默认数据文件，包括默认用户。菜品数据需要商家端手动添加。

### Q: 为什么 images 目录是空的？

A: `images/` 目录中的图片文件不会被提交到 Git 仓库（已在 .gitignore 中排除），每个开发者或部署环境需要自行添加菜品图片。这样可以：
- 减少仓库大小
- 避免提交大量二进制文件
- 允许不同环境使用不同的图片资源

**添加图片方法**：将菜品图片文件放入 `images/` 目录，然后在商家端添加菜品时使用路径格式：`images/图片文件名.jpg`

### Q: 如何添加支付功能？

A: 当前版本不包含支付功能。如需集成支付，请参考"开发说明"部分的"支付模块集成"章节。建议：
- 先搭建后端服务器处理支付请求
- 集成微信支付或支付宝支付 SDK
- 实现支付回调和订单状态同步
- 注意支付安全，不要在客户端直接调用支付 API

### Q: 如何添加扫码登录或微信登录？

A: 当前版本仅支持账号密码和人脸识别登录。如需扩展登录方式，请参考"开发说明"部分的"登录逻辑扩展"章节。需要：
- 注册相应的开放平台账号（微信开放平台等）
- 集成 OAuth 2.0 授权流程
- 实现用户信息获取和本地账号绑定
