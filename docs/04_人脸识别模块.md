# 人脸识别模块

## 概述

人脸识别模块提供人脸检测、识别和录入功能，支持 OpenCV 和 SeetaFace 两种识别方案。

## 文件列表

- `facelogindialog.h` / `facelogindialog.cpp`：人脸识别登录对话框
- `faceenrolldialog.h` / `faceenrolldialog.cpp`：人脸录入对话框
- `opencvfacerecognizer.h` / `opencvfacerecognizer.cpp`：OpenCV 人脸识别实现
- `seetafacefacerecognizer.h` / `seetafacefacerecognizer.cpp`：SeetaFace 人脸识别实现

## 功能说明

### 主要功能

1. **人脸检测**：
   - 实时检测视频流中的人脸
   - 显示检测框和提示信息
   - 支持人脸跟踪（OpenCV）

2. **人脸识别**：
   - 识别已录入的人脸
   - 返回用户名和相似度
   - 支持阈值设置

3. **人脸录入**：
   - 使用摄像头拍照
   - 检测人脸并训练模型
   - 保存人脸数据

## 使用流程

### 人脸录入流程

1. 在商家端点击"录入人脸"按钮
2. 选择或输入用户名
3. 打开人脸录入对话框
4. 启动摄像头并显示视频流
5. 实时检测人脸（显示绿色检测框）
6. 点击"拍照"按钮
7. 系统检测人脸并显示预览
8. 点击"保存"按钮
9. 保存图像文件到 `faces/` 目录
10. 训练人脸识别模型
11. 保存训练数据

### 人脸识别登录流程

1. 在商家登录对话框中选择"人脸识别登录"
2. 检查是否有人脸数据
3. 打开人脸识别对话框
4. 启动摄像头并显示视频流
5. 实时检测和跟踪人脸
6. 点击"开始识别"按钮
7. 捕获当前帧
8. 调用识别器进行识别
9. 如果识别成功且相似度超过阈值，登录成功
10. 否则显示错误提示

## 技术实现

### 视频显示优化

**问题**：`QVideoWidget` 在 Windows 上使用原生渲染，会遮挡所有子控件

**解决方案**：
- 使用 `QVideoSink` 获取视频帧
- 转换为 `QImage`
- 直接在图像上绘制检测框
- 使用 `QLabel` 显示处理后的图像

### 性能优化

1. **动态调整检测间隔**：
   - 跟踪模式：100ms（更流畅）
   - 检测模式：300ms（降低 CPU 占用）
   - SeetaFace：400ms（检测较慢）

2. **避免帧处理重叠**：
   - 使用 `m_processingFrame` 标志
   - 处理中跳过新帧

3. **优化图像处理**：
   - 仅在需要时进行格式转换
   - 使用快速缩放算法（`Qt::FastTransformation`）
   - 缓存标签尺寸

4. **跟踪优化**：
   - 跟踪比检测快，不需要格式转换
   - 跟踪失败时回退到检测
   - 连续失败 3 次后重新初始化跟踪器

### 人脸检测算法

#### OpenCV

- **Haar Cascade**：使用 `cv::CascadeClassifier`
- 模型文件：`haarcascade_frontalface_alt.xml`
- 支持多尺度检测

#### SeetaFace

- 使用 SeetaFace6Open 的 `FaceDetector`
- 支持更精确的人脸检测
- 支持关键点检测

### 人脸识别算法

#### OpenCV 基于传统特征的实现（`OpenCVFaceRecognizer`）

> 当前实现中，OpenCV 路线主要采用 **传统特征（LBP + HOG）+ 余弦相似度**，LBPH 作为后备方案，DNN 相关代码预留但默认关闭。

- **整体流程概览**：
  1. 初始化：加载 Haar 人脸检测器、初始化 LBPH 识别器（作为后备）。
  2. 训练（录入）：`train(username, faceImage)`  
     - 检测人脸 → 质量评估 → 遮挡检测 → 人脸对齐 → 提取特征（LBP+HOG）→ 特征归一化 → 更新/新增用户特征 → 保存模型。
  3. 识别：`recognize(image, confidence)`  
     - 检测人脸 → 质量评估 → 遮挡检测 → 人脸对齐 → 提取特征 → 特征归一化 → 与所有用户做余弦相似度 → 选出最相似用户 → 若不达阈值，再尝试 LBPH 后备。
  4. 模型读写：`saveModel()` / `loadModel()` 使用 JSON 存储用户名、标签和特征矩阵；LBPH 模型单独保存为 XML。  

- **人脸检测（Haar Cascade）**：
  - 使用 `cv::CascadeClassifier` 加载 `haarcascade_frontalface_alt.xml`。
  - 处理步骤：
    - QImage → `cv::Mat`。
    - 转灰度图：`cvtColor(..., COLOR_BGR2GRAY)`。
    - 直方图均衡化：`equalizeHist`，提升对比度、减弱光照影响。
    - `detectMultiScale` 进行多尺度检测，首次使用较严格参数；若失败，用更宽松参数重试。
    - 选取面积最大的人脸作为主脸，并在宽高方向上稍作扩展，保证包含完整人脸。

- **对齐与预处理**：
  - `preprocessImage`：
    - 转灰度 → 直方图均衡化 → 缩放到固定尺寸（256x256）→ 轻微高斯模糊（3x3）去噪。
  - `alignFace`：
    - 基于人脸矩形做近似对齐：
      - 若人脸太小，整体放大图像；
      - 按一定比例向四周扩展矩形并裁剪到图像范围内；
      - 最终裁剪出的人脸区域缩放为 112x112（常见的人脸识别标准尺寸）。

- **特征提取：LBP + HOG 组合**：
  - 先将对齐后的人脸转换为灰度，并做直方图均衡化。
  - **LBP（Local Binary Pattern，本地二值模式）**：
    - 对图像中每个像素，取其周围一圈 8 个邻居像素；
    - 邻居灰度 ≥ 中心灰度记为 1，反之记为 0，得到 8 位二进制数（0–255）；
    - 这一步本质上是一种 **局部二值化**，用 0/1 描述局部纹理结构。
    - 对整张 LBP 图像统计 256 维直方图（每个编码出现的频率），并做 **L1 归一化**（所有维度之和=1），得到 LBP 纹理特征。
  - **HOG（Histogram of Oriented Gradients，方向梯度直方图）**：
    - 在 112x112 的灰度图上计算梯度（类似边缘）；
    - 将图像划分为若干 cell/block，在每个小块内统计梯度方向的直方图；
    - 所有小块的直方图拼接成一个长向量，即 HOG 特征，强调轮廓和边缘结构。
  - **特征融合**：
    - 将 LBP 直方图特征与 HOG 特征在列方向上拼接（`hconcat`），得到一个大特征向量。
    - 最终特征为 `float` 类型的一维向量，用于后续相似度计算。

- **特征归一化（Normalization）**：
  - 在训练和加载特征时，使用 **L2 归一化**：
    - 设原始特征向量为 `x`，则归一化后为 `x' = x / ||x||₂`，使得向量长度为 1。
    - **注意**：`||x||₂` 中的 `||` 不是绝对值符号，而是**向量的范数（norm）**符号，表示向量的长度。`||x||₂` 是 L2 范数（欧几里得范数），计算公式为：`||x||₂ = √(x₁² + x₂² + ... + xₙ²)`，即所有元素平方和再开方。
    - 这样可以消除特征整体尺度的影响，方便使用余弦相似度进行比较。

- **相似度计算（余弦相似度）**：
  - 余弦相似度定义为：
    ```
    cos_sim(x, y) = (x · y) / (||x||₂ × ||y||₂)
    ```
    其中：
    - `x · y` 表示两个向量的点积（对应元素相乘后求和）
    - `||x||₂` 表示向量 x 的 L2 范数（即向量长度，所有元素平方和再开方）。**注意**：`||` 是范数符号，不是绝对值符号。
    - `||y||₂` 表示向量 y 的 L2 范数
  - 由于特征已经做过 L2 归一化（长度为 1），因此两个特征向量的点积 `x · y` 即为余弦相似度，范围理想情况下在 `[-1, 1]`，代码中限制在 `[0, 1]`。
  - 识别时，对输入人脸的特征与所有已保存特征逐一计算余弦相似度，找到最大值和对应用户，当最大相似度超过设定阈值（例如 0.65）即认为识别成功。

- **质量评估与遮挡检测**：
  - **质量评估（`assessFaceQuality`）**：综合考虑清晰度、亮度和人脸尺寸：
    - 清晰度：对灰度图使用拉普拉斯算子（`Laplacian`），计算结果的方差，方差越大说明纹理越丰富、越锐利；
    - 亮度：计算人脸区域的平均灰度，过暗/过亮会影响识别，转换为 0~1 的亮度评分；
    - 尺寸：人脸区域越大，信息越充足，给出尺寸评分；
    - 三者按权重组合，输出一个 0~1 的质量分数。
  - **遮挡检测（`detectOcclusion`）**：基于纹理方差和区域划分：
    - 将人脸区域分成多个关键子区（左/右眼、鼻子、嘴巴、左右脸颊等）；
    - 在每个区域内再次使用拉普拉斯算子，计算纹理方差；
    - 与全局平均方差比较，显著低于阈值的区域认为可能被遮挡（口罩、手、头发等）；
    - 综合被遮挡区域数量和左右脸颊对称性、双眼区域情况，计算一个 0~1 的遮挡比例；
    - 当遮挡比例超过一定阈值（如 10%）时，拒绝训练或识别。

- **LBPH 备用方案**：
  - 在训练时，同时把对齐后的人脸图像送入 OpenCV 的 `LBPHFaceRecognizer` 进行训练/更新；
  - 在主特征匹配失败时，尝试使用 LBPH 模型进行预测，将距离值转换为 0~1 范围的“相似度”，超过一定阈值时作为后备识别结果；
  - 该部分主要用于兼容旧版本数据和在传统特征异常时提供兜底。

#### SeetaFace DNN 识别实现（`SeetaFaceRecognizer`）

> SeetaFace 路线使用 SeetaFace6Open SDK 内部的深度神经网络，外层逻辑与 OpenCV 版本类似：检测 → 关键点 → 特征 → 余弦相似度。

- **初始化流程**：
  - 全局初始化 TenniS 运行环境（仅执行一次），确保底层推理库可用；
  - 按顺序查找模型文件：
    - 可执行文件目录下的 `models`；
    - 当前工作目录下的 `models`；
    - `SEETAFACE_ROOT` 或默认路径 `E:/share/SeetaFace/bin/model`；
  - 加载三类模型：
    - 人脸检测模型（`face_detector.csta` / `fd_2_00.dat`）；
    - 关键点模型（`face_landmarker_pts5.csta` / `pd_2_00_pts5.dat`）；
    - 识别模型（`face_recognizer.csta` / `fr_2_10.dat`）；
  - 创建 `FaceDetector`、`FaceLandmarker`、`FaceRecognizer` 三个对象，并尝试加载已有的特征库 JSON。

- **检测与关键点**：
  - 使用 `FaceDetector::detect_v2` 在整张图上检测人脸，并选择面积最大的人脸作为目标；
  - 使用 `FaceLandmarker::mark` 在该人脸区域上获取 5 个关键点（左眼、右眼、鼻尖、左/右嘴角）。

- **遮挡检测（关键点版）**：
  - 基于关键点位置判断是否被遮挡：
    - 检查每个关键点是否在合理的人脸区域内（如左眼在左上、右眼在右上等）；
    - 有效关键点数量越少，遮挡比例越高；
    - 额外检查左右眼间距是否过小（过小可能表示被遮挡或检测异常），必要时提高遮挡比例；
  - 返回 0~1 的遮挡比例，当超过某一阈值（如 10%）时，拒绝训练或识别。

- **特征提取与训练**：
  - 使用 `FaceRecognizer::GetExtractFeatureSize()` 获取特征向量维度，并调用 `Extract` 根据完整图像和关键点提取特征；
  - 对特征向量进行 **L2 归一化**（与 OpenCV 版本一致）；
  - 当用户已存在时，与旧特征做逐维平均并重新归一化，实现“多样本特征融合”，提高鲁棒性；
  - 当为新用户时，分配新的 `faceId`，保存用户名、`faceId` 及其特征；
  - 将整个特征库（用户列表、`faceId` 列表、特征矩阵）保存为 JSON 文件。

- **识别与相似度匹配**：
  - 与训练类似，先检测人脸并获取关键点，进行遮挡检测；
  - 调用 `Extract` 得到查询特征，并做 L2 归一化；
  - 遍历所有已保存特征，逐一计算 **余弦相似度**（点积），选出最大相似度和对应用户；
  - 当最大相似度超过设定阈值（例如 0.60）时，认为识别成功，并返回对应用户名和相似度。

### 名词解释

- **二值化**：
  - 将灰度值映射为 0 或 1 的过程，常见形式是和某个阈值比较（如 ≥ 阈值记为 1，< 阈值记为 0）。
  - 在本项目中，LBP 在生成二进制编码时就包含了一种“局部二值化”：邻居像素 ≥ 中心像素记为 1，否则为 0。

- **归一化（Normalization）**：
  - **L1 归一化**：使所有元素的绝对值之和为 1，常用于直方图特征（表示为频率或概率）。
  - **L2 归一化**：使向量的欧几里得长度为 1，常用于与余弦相似度配合使用，让向量的"方向"比"长度"更重要。

- **范数（Norm）**：
  - **概念**：范数是向量长度的度量，用双竖线 `||x||` 表示（注意：这不是绝对值符号，绝对值是单竖线 `|x|`，用于单个数字）。
  - **L2 范数（欧几里得范数）**：`||x||₂ = √(x₁² + x₂² + ... + xₙ²)`，即向量所有元素平方和再开方，几何意义是向量在空间中的长度。
  - **示例**：对于向量 `x = [3, 4]`，其 L2 范数为 `||x||₂ = √(3² + 4²) = √25 = 5`。
  - **归一化后的向量**：`x' = x / ||x||₂` 会将向量缩放到长度为 1，但保持方向不变。

- **直方图均衡化（Histogram Equalization）**：
  - 通过重新分配灰度值，让图像的灰度分布更均匀、对比度更高；
  - 对人脸识别来说，可以缓解光照变化带来的影响，让特征更加稳定。

- **LBP（Local Binary Pattern，本地二值模式）**：
  - 一种描述局部纹理的算子，通过比较邻域像素与中心像素的大小关系，生成二进制编码；
  - 对每个像素生成的 LBP 值在整图上统计直方图，就得到了画面纹理模式的分布特征。

- **HOG（Histogram of Oriented Gradients，方向梯度直方图）**：
  - 通过计算图像局部区域的梯度方向直方图，描述边缘和轮廓结构；
  - 常用于行人检测、人脸检测等任务，具有一定的光照不变性。

- **余弦相似度（Cosine Similarity）**：
  - 度量两个向量在方向上的相似程度，几何意义是两向量夹角的余弦值；
  - 在特征都经 L2 归一化的前提下，点积就是余弦相似度，值越接近 1 表示越相似。

- **拉普拉斯算子（Laplacian）**：
  - 一种二阶微分算子，常用于检测图像中的高频成分（纹理、边缘）；
  - 在本项目中，通过计算拉普拉斯结果的方差来衡量图像清晰度或纹理丰富程度。

- **遮挡检测（Occlusion Detection）**：
  - 判断人脸关键区域（如眼睛、鼻子、嘴巴、脸颊等）是否被遮挡的过程；
  - OpenCV 路线通过纹理方差和区域划分来估计；
  - SeetaFace 路线通过关键点位置和相对关系来估计。

## 配置说明

### OpenCV 配置

- 默认路径：`E:/share/opencv452`
- 在 `CMakeLists.txt` 中配置 `OPENCV_ROOT`
- 可选模块：
  - `opencv_contrib`：提供 LBPH 和跟踪功能
  - `opencv_tracking`：提供跟踪功能

### SeetaFace 配置

- 默认路径：`E:/share/SeetaFace`
- 环境变量：`SEETAFACE_ROOT`
- 模型文件路径：`{SEETAFACE_ROOT}/bin/model`
- 必需模型文件：
  - `face_detector.csta` 或 `fd_2_00.dat`
  - `face_landmarker_pts5.csta` 或 `pd_2_00_pts5.dat`
  - `face_recognizer.csta` 或 `fr_2_10.dat`

## 注意事项

1. **摄像头权限**：确保程序有摄像头访问权限
2. **模型文件**：确保模型文件路径正确
3. **识别阈值**：
   - OpenCV：默认 0.6（60% 相似度）
   - SeetaFace：默认 0.55（55% 相似度）
4. **性能**：实时检测会消耗 CPU 资源，建议在性能较好的设备上使用
5. **光照条件**：良好的光照条件有助于提高识别准确率
