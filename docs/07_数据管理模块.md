# 数据管理模块

## 概述

数据管理模块提供 JSON 格式的数据持久化功能，包括用户数据、菜品数据、订单数据和人脸数据的读写操作。

## 文件列表

- `jsonhelper.h` / `jsonhelper.cpp`：JSON 数据管理工具类

## 功能说明

### 主要功能

1. **用户数据管理**：
   - 保存/加载用户账号和密码
   - 检查用户是否存在
   - 初始化默认用户

2. **菜品数据管理**：
   - 保存/加载菜品列表
   - 支持菜品增删改查

3. **订单数据管理**：
   - 保存/加载订单列表
   - 支持订单查询和更新

4. **人脸数据管理**：
   - 保存/加载人脸数据
   - 检查人脸数据是否存在

## 使用流程

### 用户数据操作

#### 保存用户

```cpp
bool success = JsonHelper::saveUser("user1", "123456");
```

#### 加载用户

```cpp
QString password;
if (JsonHelper::loadUser("user1", password)) {
    // 使用密码
}
```

#### 检查用户是否存在

```cpp
if (JsonHelper::userExists("user1")) {
    // 用户存在
}
```

### 菜品数据操作

#### 保存菜品列表

```cpp
QVector<Dish> dishes;
// ... 填充菜品数据
bool success = JsonHelper::saveDishes(dishes);
```

#### 加载菜品列表

```cpp
QVector<Dish> dishes;
if (JsonHelper::loadDishes(dishes)) {
    // 使用菜品数据
}
```

### 订单数据操作

#### 保存订单

```cpp
Order order("0210001", "顾客", items, OrderStatus::Pending);
bool success = JsonHelper::saveOrder(order);
```

#### 加载订单列表

```cpp
QVector<Order> orders;
if (JsonHelper::loadOrders(orders)) {
    // 使用订单数据
}
```

#### 更新订单状态

```cpp
bool success = JsonHelper::updateOrderStatus("0210001", "Processing");
```

### 人脸数据操作

#### 保存人脸数据

```cpp
bool success = JsonHelper::saveFaceData("user1", "faces/user1.jpg");
```

#### 加载人脸数据

```cpp
QString facePath = JsonHelper::loadFaceData("user1");
```

#### 检查是否有人脸数据

```cpp
if (JsonHelper::hasAnyFaceData()) {
    // 有人脸数据，可以使用人脸识别
}
```

## 数据初始化

### 默认用户初始化

程序首次运行时，如果没有 `user.json` 文件，会自动创建默认用户：

- **账号**：`admin`
- **密码**：首次运行时提示设置
- **类型**：商家

### 默认菜品初始化

如果 `dishes.json` 文件不存在，程序会创建一个空文件，等待商家添加菜品。

## 错误处理

### 文件操作错误

- **文件不存在**：创建新文件
- **文件读取失败**：返回 false，使用空数据
- **文件写入失败**：返回 false，数据未保存
- **JSON 解析失败**：返回 false，使用空数据

### 数据验证

- **必需字段缺失**：返回 false
- **数据类型错误**：返回 false
- **数据格式错误**：返回 false

## 文件位置

### 默认文件路径

- `user.json`：用户数据文件
- `dishes.json`：菜品数据文件
- `orders.json`：订单数据文件
- `faces/`：人脸图片目录

### 文件位置说明

所有数据文件存储在程序运行目录下，与可执行文件同级。

## 数据备份

### 当前实现

- 无自动备份机制

### 建议改进

## 性能考虑

### 文件读写

- 每次操作都完整读写文件
- 数据量大时可能影响性能
- 建议使用数据库（如 SQLite）替代

### 内存使用

- 所有数据加载到内存
- 适合小规模数据
- 大规模数据建议使用数据库

## 安全性

### 当前实现

- 密码明文存储
- 无数据加密
- 无访问控制

### 建议改进

1. **密码加密**：
   - 使用哈希算法（如 bcrypt）
   - 存储密码哈希值

2. **数据加密**：
   - 敏感数据加密存储
   - 使用 AES 加密

3. **访问控制**：
   - 文件权限控制
   - 数据访问权限验证

## 注意事项

1. **数据一致性**：多线程访问时需要注意同步
2. **文件锁定**：Windows 上文件被占用时无法写入
3. **路径问题**：使用相对路径，确保文件位置正确
4. **编码问题**：JSON 文件使用 UTF-8 编码
5. **数据迁移**：修改数据结构时需要兼容旧数据
