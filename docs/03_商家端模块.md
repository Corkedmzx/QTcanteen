# 商家端模块

## 概述

商家端模块提供商家管理功能，包括菜品管理、订单管理、人脸录入和网络服务器管理。

## 文件列表

- `merchantmainwindow.h` / `merchantmainwindow.cpp`：商家端主窗口
- `merchantmainwindow.ui`：UI 设计文件

## 功能说明

### 主要功能

1. **菜品管理**：
   - 添加新菜品
   - 编辑现有菜品
   - 删除菜品
   - 保存菜品数据

2. **订单管理**：
   - 查看所有订单
   - 查看订单详情
   - 确认完成订单（每行订单后提供"确认完成"按钮）
   - 订单完成后自动发送通知给客户端
   - 删除已完成订单
   - 自动刷新订单列表

3. **账户管理**：
   - 创建新账户（用于分店经理登录）
   - 查看已创建账户列表
   - 显示账户创建时间

4. **人脸录入**：
   - 为指定用户录入人脸
   - 使用摄像头拍照
   - 人脸检测和训练

5. **网络服务器**：
   - 启动 TCP 服务器（端口 8888）
   - 接收客户端订单
   - 广播菜品更新通知
   - 广播订单完成通知

## 使用流程

### 菜品管理流程

1. **添加菜品**：
   - 点击"添加菜品"按钮
   - 输入菜品信息（名称、分类、价格、描述、图片路径）
   - 点击"保存"按钮
   - 系统自动生成菜品ID并保存

2. **编辑菜品**：
   - 在菜品列表中选择菜品
   - 点击"编辑菜品"按钮
   - 修改菜品信息
   - 点击"保存"按钮

3. **删除菜品**：
   - 在菜品列表中选择菜品
   - 点击"删除菜品"按钮
   - 确认删除操作

4. **保存菜品**：
   - 点击"保存菜品"按钮
   - 将所有菜品数据保存到 `dishes.json`
   - 广播菜品更新通知给所有客户端

### 订单管理流程

1. **查看订单**：
   - 订单列表自动显示所有订单
   - 显示订单号、用户名、状态、总金额、创建时间
   - 使用不同颜色标识不同状态

2. **查看订单详情**：
   - 双击订单行
   - 打开订单详情对话框
   - 显示订单所有信息

3. **确认完成订单**：
   - 在订单列表中，每行订单后有一个"确认完成"按钮
   - 点击"确认完成"按钮
   - 确认操作后，订单状态更新为"已完成"
   - 系统自动发送订单完成通知给所有连接的客户端
   - 客户端收到通知后会显示提示消息

4. **自动刷新**：
   - 每 3 秒自动刷新订单列表
   - 显示最新订单数据

### 账户管理流程

1. **创建新账户**：
   - 在"账户下发"标签页中
   - 输入新账号和密码
   - 确认密码
   - 点击"创建账户"按钮
   - 系统保存账户并显示创建时间

2. **查看账户列表**：
   - 在"账户下发"标签页中
   - "已创建账户列表"区域显示所有非商家账户
   - 显示账号和创建时间

### 人脸录入流程

1. 在"账户下发"标签页中
2. 选择要录入人脸的账户
3. 点击"录入人脸"按钮
4. 打开人脸录入对话框
5. 使用摄像头拍照
6. 检测人脸并训练模型
7. 保存人脸数据

### 网络服务器流程

1. **启动服务器**：
   - 窗口创建时自动创建 `NetworkServer` 对象
   - 自动启动服务器（端口 8888）
   - 在状态栏显示启动状态

2. **接收订单**：
   - 监听客户端连接
   - 接收订单数据
   - 发出 `newOrderReceived` 信号
   - 发送订单确认消息

3. **广播更新**：
   - 菜品更新时广播通知
   - 所有连接的客户端接收更新

## 数据交互

### 使用的数据管理接口

- `JsonHelper::loadDishes()`：加载菜品数据
- `JsonHelper::saveDishes()`：保存菜品数据
- `JsonHelper::loadOrders()`：加载订单数据
- `JsonHelper::saveOrder()`：保存订单数据
- `JsonHelper::loadUsers()`：加载用户列表（用于人脸录入）

### 数据文件

- `dishes.json`：菜品数据
- `orders.json`：订单数据
- `user.json`：用户数据

## UI 组件

4. **状态栏**：
   - 显示网络服务器状态
   - 显示操作提示信息

## 订单状态管理

### 订单状态

- **待处理（Pending）**：新订单，等待处理
- **已完成（Completed）**：订单已完成

> 当前实现中，订单从 **待处理** 直接流转到 **已完成**。`Processing` / `Cancelled` 状态仅作为兼容旧数据的保留值，界面上不再单独区分。

### 状态颜色标识

- 待处理：浅红色背景
- 已完成：浅绿色背景

## 网络通信

### 服务器配置

- 默认端口：8888
- 监听地址：`QHostAddress::Any`（所有网络接口）
- 支持多客户端连接

### 消息处理

1. **接收消息类型**：
   - `order`：订单数据
   - `checkout`：结账请求
   - `service`：服务请求

2. **发送消息类型**：
   - `order_confirm`：订单确认
   - `checkout_success`：结账成功
   - `dish_update`：菜品更新通知
   - `order_completed`：订单完成通知（包含订单号和提示消息）

## 定时刷新机制

### 订单刷新

- 刷新间隔：3 秒
- 使用 `QTimer` 实现
- 自动加载最新订单数据
- 更新订单列表显示

## 注意事项

1. **网络服务器**：确保端口 8888 未被占用
2. **订单状态**：状态更新会立即保存到文件
3. **菜品保存**：修改菜品后需要点击"保存菜品"按钮
4. **人脸录入**：需要摄像头设备支持
5. **数据同步**：网络断开时，订单仍会保存到本地文件
