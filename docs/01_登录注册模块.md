# 登录注册模块

## 概述

登录注册模块提供用户身份验证和账号管理功能，包括登录窗口、注册窗口和商家登录对话框。

## 文件列表

- `loginwindow.h` / `loginwindow.cpp`：登录窗口
- `registerwindow.h` / `registerwindow.cpp`：注册窗口

## 功能说明

### 登录窗口 (LoginWindow)

#### 主要功能
1. **入口选择**：
   - 点餐入口：直接进入客户端（无需登录）
   - 商家登录：打开商家登录对话框

2. **商家登录对话框**：
   - 账号密码登录
   - 人脸识别登录
   - 更改账号/密码功能

#### 核心方法

```cpp
void onOrderClicked()           // 点餐入口处理
void onMerchantLoginClicked()   // 商家登录入口处理
```

#### 信号

```cpp
void loginSuccess(const QString &username, bool isMerchant)
```

### 注册窗口 (RegisterWindow)

#### 主要功能
1. **用户注册**：
   - 输入用户名
   - 输入密码
   - 确认密码
   - 验证输入有效性

#### 核心方法

```cpp
void onRegisterClicked()        // 注册按钮处理
void onCancelClicked()          // 取消按钮处理
bool validateInput()            // 输入验证
```

#### 信号

```cpp
void registerSuccess(const QString &username)
```

## 使用流程

### 商家登录流程

1. 用户点击"商家登录"按钮
2. 弹出商家登录对话框
3. 选择登录方式：
   - **账号密码登录**：
     - 输入账号和密码
     - 验证账号是否存在
     - 验证密码是否正确
     - 登录成功后发出 `loginSuccess` 信号
   
   - **人脸识别登录**：
     - 检查是否有人脸数据
     - 打开人脸识别对话框
     - 识别成功后验证账号
     - 登录成功后发出 `loginSuccess` 信号

4. 登录成功后打开商家端窗口

### 用户注册流程

1. 打开注册窗口
2. 输入用户名、密码和确认密码
3. 验证输入：
   - 用户名不能为空
   - 密码不能为空
   - 两次密码输入必须一致
   - 用户名不能已存在
4. 保存用户信息到 `user.json`
5. 发出 `registerSuccess` 信号

### 更改账号/密码流程

1. 在商家登录对话框中点击"更改账号/密码"
2. 输入当前账号和密码
3. 输入新账号（可选）和新密码
4. 确认新密码
5. 验证：
   - 当前账号密码正确
   - 新密码不为空
   - 两次新密码输入一致
   - 新账号不冲突（如果修改账号）
6. 保存新账号密码

## 数据交互

### 使用的数据管理接口

- `JsonHelper::loadUser()`：加载用户信息
- `JsonHelper::saveUser()`：保存用户信息
- `JsonHelper::userExists()`：检查用户是否存在
- `JsonHelper::hasAnyFaceData()`：检查是否有人脸数据

### 数据文件

- `user.json`：存储用户账号和密码

## UI 设计

### 登录窗口
- 窗口大小：450x360
- 非模态窗口，支持独立最小化
- 包含标题、功能说明和按钮区域

### 商家登录对话框
- 窗口大小：420x320
- 模态对话框
- 包含账号、密码输入框和多个操作按钮

### 注册窗口
- 窗口大小：400x350
- 模态对话框
- 包含用户名、密码、确认密码输入框

## 注意事项

1. **安全性**：密码以明文存储在 JSON 文件中，生产环境建议加密存储
2. **账号管理**：商家账号和普通用户账号共用同一存储文件
3. **人脸识别**：需要先录入人脸数据才能使用人脸识别登录
4. **窗口管理**：登录窗口不隐藏，允许同时打开多个窗口进行测试
